# Taste（口味）PRD v0.1

## 1. Product Overview

### 1.1 Product Name
- 中文名：**口味**
- 英文名：**Taste**

### 1.2 Product Title
**一口食物，一口生活。**

### 1.3 Product Slogan
Taste 既代表真实的食物味道，也代表虚缈而细腻的生活味道。  
它记录的不是“今天吃了什么”这么简单，而是“那一刻我过得怎么样”。

### 1.4 Product Positioning
Taste 是一款以**饮食记录**为入口，连接**情绪、生活碎片、关系记忆**的轻私密生活记录小程序。

它不是一个：
- 健身减脂打卡工具
- 菜谱教学平台
- 公开社交平台
- 单纯的心情日记本

它更像是一个：
- 可以保存不想发朋友圈内容的空间
- 允许记录不好看但真实生活的地方
- 可以通过时间回看变化的生活档案

## 2. Background & Problem Statement

### 2.1 Background
很多人会拍下吃饭、做饭、生活片段的照片，但这些内容常常面临几个问题：

1. **不适合发朋友圈 / 小红书**
   - 有些内容太私密
   - 有些内容不够“精致”
   - 有些照片只是想留给自己看

2. **聊天记录容易被冲掉**
   - 某些珍贵时刻只存在于微信聊天里
   - 过一段时间后很难再次回看
   - 缺少系统化保存与归档

3. **普通记录类 App 容易同质化**
   - 只有打卡，没有温度
   - 只有输入，没有回看价值
   - 只有“记录功能”，没有“生活感”

### 2.2 Core Problem
用户缺少一个轻私密、有温度、可长期回看、以吃饭和生活片段为入口的记录空间。

---

## 3. Product Goals

### 3.1 Short-Term Goal (MVP)
在微信小程序中实现一个最小可用版本，支持用户：
- 新建生活记录
- 记录食物 / 图片 / 文字 / 心情
- 浏览历史记录
- 使用月历回看有记录的日期
- 查看某条记录详情

### 3.2 Mid-Term Goal
在 MVP 跑通后逐步扩展：
- 菜品历史回看
- 情绪筛选
- 月报回顾
- 收藏 / 珍藏
- 云同步

### 3.3 Long-Term Goal
打造一个兼具：
- 生活记录
- 情绪留存
- 时间回看
- 关系记忆  
的轻私密陪伴型产品。

## 4. Target Users

### 4.1 Core Users
#### User Type A：不喜欢公开表达，但想保留生活的人
- 不常发朋友圈
- 不想经营社交平台形象
- 但会被生活中的小瞬间打动
- 希望留住自己的真实生活

#### User Type B：愿意轻量记录饮食和情绪的人
- 喜欢拍吃饭照片
- 会对某一餐赋予情绪意义
- 不一定写长日记，但愿意写一句话

#### User Type C：有私密记录需求的人
- 想保存和对象的生活碎片
- 想记录不好看但珍贵的照片
- 希望记录只有自己能看到

### 4.2 Example User Personas

#### Persona 1：独居女生
- 年龄：22-28
- 特点：喜欢记录，但不喜欢公开发社交平台
- 需求：记录做饭、外食、情绪、生活状态

#### Persona 2：情侣用户中的一方
- 年龄：20-30
- 特点：喜欢保存和对象的相处细节
- 需求：记录一起吃饭、旅行、丑照、生活片段

#### Persona 3：情绪敏感型用户
- 年龄：20-30
- 特点：会被某一顿饭、某一时刻触动
- 需求：通过轻量记录保留那一刻的生活感受

## 5. Core Value Proposition

Taste 的核心价值不在于“记录得多详细”，而在于：

### 5.1 Preserve Real Life
保存那些真实但不够精致的生活瞬间。

### 5.2 Keep Private but Precious Moments
给不适合公开表达、却很珍贵的内容一个长期留存的空间。

### 5.3 Make Changes Visible
让用户通过时间回看，看到自己在饮食、情绪、生活中的变化。

### 5.4 Build a Gentle Personal Archive
构建一个温柔、轻私密、不需要表演的个人生活档案。

## 6. Core Scenarios

### Scenario 1: Cooking Record
用户自己做了一顿饭，拍照并记录：
- 今天做了什么
- 味道如何
- 当时心情如何

### Scenario 2: Eating Out with Emotion
用户在外面吃饭，但当天情绪很差，没吃完喜欢的牛肉粉，于是拍照记录：
- 这一餐
- 没吃完的状态
- 自己当时的心情

### Scenario 3: Precious Couple Moments
用户和对象吃饭时拍下了一张不好看但珍贵的照片，不想发社交平台，但想留住。

### Scenario 4: Looking Back Over Time
用户在月底通过月历和历史记录回看：
- 这个月吃了什么
- 哪些天心情差
- 哪些菜做得越来越好
- 哪些日子最值得回忆

## 7. Product Scope

### 7.1 MVP Scope (v0.1)
MVP 第一版仅实现最核心闭环：

#### Included
- 首页
- 新建记录页
- 回顾列表页
- 日历页（月视图基础版）
- 记录详情页
- 本地数据存储
- 基础记录类型与心情标签

#### Excluded
- 用户登录
- 云同步
- 收藏功能
- 月报 / 年报
- 菜品成长页
- AI 自动总结
- 双人共享空间
- 搜索功能
